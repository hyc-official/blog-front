如何优雅地使用 GitHub 搞出自己的博客？

巨佬们大多都有自己的博客，但是想要一个动态的博客空间，可是要钱的！所以就有很多人使用静态博客 + GitHub Pages 托管。

好东西们

GitHub 的一些产品为本文提供了理论与实践基础：

  • GitHub Pages
  • GitHub Actions
  • GitHub.dev

另外，还有 Lightly 也强力支持了本方法。

GitHub Pages 托管网站

在某谷，经常有时可以看见某些大佬的主页有这样的链接：My Blog，指向一个形如 xxx.github.io 的网站，这就是 GitHub Pages。

GitHub Pages 为用户提供静态网页托管服务，但是增加了 MD 转 HTML，可设置主题的 Jekyll。

GitHub Actions 构建项目

不知道各位有没有用过 Linux 的终端，如果实在没用过，Windows 的 CMD 也能来凑个数。

GitHub Actions，说简单点，就是给你提供了一个终端，你可以用它来进行你的项目的构建。

更灵活地控制部署

但是需要牺牲你的一些时间。

我们可以建立一个新的分支 build，把 Actions 设为 Push 到 build 时才执行构建，然后每次修改好后 PR 到 build 即可。

GitHub.dev 在线修改

GitHub.dev 是 GitHub 的在线编辑，相当于网页精简版 VSCode + GitHub 扩展。

在没有配置环境的电脑上，使用 GitHub.dev 实在是一个好的选择。

正文开始

下文中 GH 为 GitHub 的简写。

博客系统

GH Pages 内置的 Jekyll 有许多博客的主题，但是可自定义度不是那么高，所以被更多人采用的是基于 Node.js 的 Hexo（https://hexo.io）。Hexo 有许多主题，光是被官网收录的就有一大堆，更别说那些没有收录的了（如 Argon）。

于是在下文中，我们使用 Hexo 来搭建博客。

在线编辑

注意：这里介绍使用 GitHub 帐号注册 Lightly 的方法，你也可以使用其他方法，这里就不再提及。如果你使用的是其他方法，请酌情跳过本步骤。

本步骤请先注册 GitHub 账号。

打开 Lightly 官网（https://lightly.teamcode.com），点击 在线使用，跳转到登录页。

点击下面的 GitHub 小图标，跳转到 GitHub 授权页面，点击 Authorize xxx，授权登录 Lightly。


然后会回到 Lightly 主页，再次点击 在线使用，进入 Lightly 面板。

点击 +新建项目，创建一个 Node.js 项目，命名随意。

随后就是等待项目初始化。

项目初始化完成后，安装 Hexo 并初始化博客。由于 Lightly 似乎在根目录下放了点东西，我们只能在子目录中初始化。

接下来，你可以配置你的博客，这里我就不演示了。Lightly 的 Terminal 就是终端，可以使用终端命令。

接下来，部署到 GH Pages 这种官方文档都写了的东西我就不细讲了。

居安思危

毕竟 Lightly 不是某里、某讯、微硬之类的大厂搞的，我们也不得不想一想这个问题：万一 Lightly 倒了怎么办？

于是想到，我们可以把 Hexo 源文件扔到 GitHub 上，聪明！

Lightly 的机子，GitHub 的拉取和推送还是比较快的，命令行中也集成了 Git。至于 Git 操作我也就不多说了。
